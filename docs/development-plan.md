# 网络测试工具箱开发规划

> 基于 29 张 UI 设计稿整理，使用 Python 技术栈，分阶段实现。

---

## 一、技术方案

### 1.1 技术栈选型

| 层级 | 技术 | 说明 |
|------|------|------|
| UI 框架 | PySide6 | Qt 的 Python 绑定，现代 UI，布局系统强大，比 tkinter 更适合复杂界面 |
| 系统信息 | psutil | 获取 CPU/内存/磁盘/网络/进程信息 |
| 网络探测 | scapy | 数据包抓包、协议分析、网络扫描 |
| HTTP 请求 | requests | 外网测速、IP 信息查询 |
| 串口通信 | pyserial | 串口调试功能 |
| SSH | paramiko | 远程终端 SSH 连接 |
| 图表 | matplotlib + PyQtGraph | 仪表盘图表、实时数据可视化 |
| 打包 | PyInstaller | 打包为 exe，目标体积约 80M |
| 图标 | Qt SVG | 使用 SVG 矢量图标 |

### 1.2 项目架构

```
project_03_network-testing-toolkit/
├── src/
│   ├── main.py                 # 入口
│   ├── app.py                  # 主应用类

```

## 三、详细难点分析

### 3.1 权限问题（高难度）

| 功能 | 权限要求 | 实现方式 |
|------|----------|----------|
| 防火墙配置 | 管理员 | `netsh advfirewall` 命令 |
| 安全日志审计 | 管理员 | `wevtutil` 命令 / `win32evtlog` |
| 安全自测（部分项） | 管理员 | 注册表读取、组策略查询 |
| 修改 3389 端口 | 管理员 | 注册表修改 + 重启终端服务 |
| 路由表增删 | 管理员 | `route add/delete` 命令 |
| 数据包抓包 | 管理员 | scapy + npcap |
| 进程管理 | 管理员 | psutil（部分操作） |

**难点**：需要检测当前运行权限，无权限时提示用户重新以管理员身份运行。

### 3.2 性能问题（中难度）

| 功能 | 性能挑战 | 解决方案 |
|------|----------|----------|
| 端口扫描 | 大量并发连接 | 线程池 + 超时控制 |
| 主机发现 | 全网段扫描 | 异步 IO + 线程池 |
| 摄像头扫描 | 多协议探测 | 协程/线程池 + 超时 |
| 会话数测试 | 大量并发连接 | 连接池 + 异步处理 |
| 实时流量分析 | 高速数据包解析 | scapy 回调 + 数据聚合 |
| 仪表盘刷新 | 实时数据更新 | 定时器 + 批量更新 |

**难点**：平衡扫描速度与系统资源占用，避免 UI 卡顿。

### 3.3 协议实现（高难度）

| 协议/服务 | 复杂度 | 依赖库 |
|-----------|--------|--------|
| TFTP 服务器 | 中 | 自定义实现 |
| FTP 服务器 | 高 | pyftpdlib |
| DHCP 服务器 | 高 | scapy 或自定义 |
| Syslog 服务器 | 低 | socket |
| ONVIF 发现 | 中 | onvif-zeep |
| SSH 客户端 | 中 | paramiko |
| RDP 客户端 | 高 | 调用 mstsc 或 freerdp |

**难点**：DHCP 协议复杂，需要完整实现租约管理；RDP 需要调用系统工具或使用第三方库。

### 3.4 系统依赖（中难度）

| 功能 | 依赖 | 安装方式 |
|------|------|----------|
| 数据包抓包 | npcap/winpcap | 用户安装或静默安装 |
| 串口调试 | pyserial | pip |
| WiFi 分析 | pywifi | pip（Windows 专用） |
| 图表 | matplotlib | pip |
| SSH | paramiko | pip |

**难点**：scapy 需要 npcap 驱动，打包时需处理依赖安装。

### 3.5 UI 复杂度（中难度）

| 组件类型 | 挑战 |
|----------|------|
| 多标签页 | Ping 测试有 5 个标签页，连接测试有 11 个标签页 |
| 实时图表 | CPU/内存/网络吞吐需要实时折线图 |
| 终端模拟 | SSH/Telnet/串口终端需要 VT100 支持 |
| 表格显示 | 大量扫描结果需要高性能表格控件 |
| 动态布局 | 不同页面布局差异大 |

**难点**：PySide6 的 QTableView 性能有限，大量数据需要分页或虚拟滚动。

### 3.6 打包问题（中难度）

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 体积过大 | scapy/matplotlib 依赖多 | PyInstaller 排除不必要模块 |
| npcap 依赖 | 需要系统驱动 | 安装包中包含 npcap 安装程序 |
| 图标问题 | Windows 应用图标 | PyInstaller 配置图标 |
| 权限提示 | 部分功能需要管理员 | 应用清单配置 |

**难点**：目标 80M，需要仔细裁剪依赖。

## 四、风险点

1. **scapy 兼容性**：scapy 在 Windows 下需要 npcap，且版本兼容性可能有问题
2. **管理员权限**：部分功能必须管理员权限，用户体验可能受影响
3. **系统命令依赖**：依赖 `ping`、`tracert`、`netsh` 等命令，不同 Windows 版本可能有差异
4. **防火墙拦截**：Windows Defender 可能拦截网络扫描行为
5. **打包体积**：scapy + matplotlib 打包后体积可能超 80M，需要优化
6. **串口设备**：串口调试依赖物理硬件，测试环境可能受限
7. **WiFi 分析**：pywifi 在 Windows 10/11 上行为可能不一致
8. **摄像头扫描**：
   - **ONVIF协议**：依赖zeep库，体积较大（~10MB），需考虑排除xml相关模块
   - **品牌识别**：私有协议无法100%识别，扫描结果仅供参考
   - **认证设备**：需要用户名密码的设备无法获取详细信息
   - **扫描性能**：/24网段254个IP全扫描需30秒-2分钟，需优化并发
   - **zeep依赖**：lxml + zeep 打包后可能增加5-10MB体积
9. **iperf3集成**：打包时附带iperf3.exe，体积增加约1-2MB
10. **连接测试模块**：
    - **DNS查询**：dnspython库体积小，无兼容性问题
    - **Whois查询**：python-whois对部分新顶级域名支持不完整，需自行解析
    - **SSL检测**：使用ssl标准库，Win7下TLS版本可能受限
    - **Telnet交互**：需要实现VT100终端模拟，复杂度较高
    - **MTU测试**：依赖系统ping命令的-f参数，不同Windows版本可能有差异
    - **HTTP Headers**：requests库足够，但需处理重定向和编码问题

## 五、开发建议

1. **先做核心功能**：Ping、路由追踪、端口扫描、子网计算是最常用的，优先完成
2. **线程池管理**：所有耗时操作必须放在线程中，使用信号槽更新 UI
3. **日志系统**：统一日志格式，方便调试和问题排查
4. **配置持久化**：用户配置（目标主机、线程数、超时等）保存到 JSON 文件
5. **错误处理**：每个功能模块要有完善的错误捕获和提示
6. **测试驱动**：每个工具模块单独写单元测试
7. **打包策略**：早期就开始测试打包，避免后期才发现体积问题

## 八、新增难点与风险（基于经验补充）

### 8.1 Windows 7 兼容性问题

| 问题 | 影响 | 解决方案 |
|------|------|----------|
| Python 版本 | Python 3.10+ 不支持 Win7 | 使用 Python 3.9（最后支持 Win7 的版本） |
| TLS 支持 | Win7 原生 TLS 1.2 支持有限 | requests 需要额外配置 |
| scapy | scapy 在 Win7 上可能需要旧版 npcap | 准备兼容方案 |
| matplotlib | 最新版 matplotlib 可能不兼容 Win7 | 使用兼容版本 |
| psutil | 部分系统 API 在 Win7 上行为不同 | 条件判断处理 |

### 8.2 PyInstaller 打包经验（来自历史踩坑）

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| PermissionError | 输出文件被占用或权限不足 | 输出目录带时间戳，构建前清理旧文件 |
| 中文乱码 | bat 脚本编码不一致 | 脚本开头设置 `chcp 65001` |
| No module named pyinstaller | 环境混用（Anaconda/venv/系统Python） | 使用项目内 venv 的完整路径执行 |
| 体积过大 | scapy/matplotlib 依赖过多 | 使用 spec 文件排除不必要模块 |
| UPX 压缩问题 | UPX 与某些模块不兼容 | 禁用 UPX 或使用特定版本 |

### 8.3 权限管理经验（来自历史踩坑）

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 用户权限感知 | 用户不知道需要管理员权限 | 启动时检测权限，无权限时弹窗提示 |
| 权限检测失败 | 不同 Windows 版本检测方式不同 | 使用多种方式检测（ctypes + 命令行） |
| 自动提权干扰 | 用户不希望每次都弹 UAC | 不自动提权，仅提示用户手动操作 |
| 功能部分可用 | 部分功能需要权限，部分不需要 | 每个功能独立检测权限 |

### 8.4 Qt 导入问题（来自历史踩坑）

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| NameError | 使用 Qt 组件但未导入 | 文件顶部集中维护导入清单，新增控件时同步补齐 |
| 导入冗余 | 导入大量未使用的组件 | 按需导入，避免 `from PySide6.QtWidgets import *` |
| 版本兼容性 | PySide6 版本差异 | 固定使用特定版本，避免升级带来的兼容性问题 |

### 8.5 系统命令差异

| 命令 | Win7 与 Win10/11 差异 | 处理方式 |
|------|------------------------|----------|
| `ping` | 参数基本一致 | 通用 |
| `tracert` | 参数基本一致 | 通用 |
| `netsh advfirewall` | 部分子命令差异 | 条件判断 |
| `wevtutil` | 参数基本一致 | 通用 |
| `route` | 参数基本一致 | 通用 |
| `ipconfig` | 参数基本一致 | 通用 |
